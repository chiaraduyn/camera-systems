Scenario: Set up a Geovison camera that is directly connected to your Windows PC via Ethernet. 

----
NOTE: This file contains original raw notes. A cleaned, GitHub-friendly version of these instructions has been moved into `docs/geovision_setup.md` and per-camera templates under `cameras/`. See `README.md` and `HIGHLIGHTS.md` for an overview.


You'll need to know:
Your computers IP: 192.168.13.50 (from ipconfig)
Subnet Mask: 255.255.255.0
Camera IP: Assign manually if not already set (e.g., 192.168.13.100)


**Note** Both devices must be on the same subnet to communicate.





1. Install GeoVision Software
1. Go to GeoVision official website → GV‑IP Camera software download
2. Install GV‑EasySetup / WebConfig Plugin (required for GV‑TBL8804 web interface)
3. Follow installation prompts: accept license, install drivers, allow firewall permissions.
4. Restart laptop if prompted.


    2. Firmware Update (Optional but Recommended)
1. Log into camera via web interface (initial IP or default)
2. Go to: Maintenance → Firmware Upgrade
3. Upload latest .bin firmware file (downloaded from GeoVision)
4. Wait for the camera to restart
5. Confirm camera version in System Info
________________


3. Initial Network Setup (Direct to PC)


1. Connect camera via Ethernet:
   * Direct cable → your Windows PC (in my case HP laptop) Ethernet Port
   * Or via PoE injector / small switch → Windows PC
2. Set static IP addresses (if needed):
   * Example:
      * Camera: 192.168.13.100
      * PC Ethernet: 192.168.13.50
      * Subnet mask: 255.255.255.0
      * Gateway: leave blank for direct connection
3. Confirm access:
   * Open browser → http://192.168.13.100
   * Log in using GeoVision credentials



________________


4. FileZilla Server Setup on PC
Make sure you have FileZilla installed and a folder for camera uploads:
e.g., C:\Users\Guest-UH\Desktop\Geovision_FTP
Step 1: Create FTP User
1. Open FileZilla Server → Edit → Users

2. Click Add → Username: geovision → OK

3. Set password: geovision123

4. Enable Password Authentication

Step 2: Map Mount Point
   1. In Users → Shared Folders → Add

   2. Set Virtual Path: /

   3. Set Native Path: C:\Users\Guest-UH\Desktop\Geovision_FTP

   4. Enable permissions:

      * Read ✔

      * Write ✔

      * Apply permissions to subdirectories ✔

      * Create native directory if it does not exist ✔

Step 3: Confirm Server Running
         * FileZilla must show Listening on port 21

         * Allow through Windows Firewall if needed

         * Do NOT use 127.0.0.1 — the camera must connect using the real IP 192.168.13.50

________________


3️⃣ Camera FTP Page Configuration
Page Location: GeoVision Web Interface → Network → FTP
Field
	Value
	Notes
	Server IP
	192.168.13.50
	Laptop Ethernet IP
	Port No.
	21
	Default FTP port
	Username
	geovision
	Matches FileZilla user
	Password
	geovision123
	Match password set in FileZilla
	Upload Images
	✔ Photo
	Only upload images, not video for now
	File Path
	/ or blank
	Root of mapped folder
	File Name
	optional template
	e.g., CAM1_%Y-%m-%d_%H-%M-%S.jpg
	Overwrite
	OFF
	Prevents overwriting files
	Test
	Click after entering info
	Watch FileZilla logs for:
	

	

	Connection received → STOR image.jpg → OK
	Tip: Only check Upload Images. Leave Video Upload and Recording unchecked for local testing.
________________


4️⃣ Motion Detection Rule Setup
Page Location: GeoVision Web Interface → Event → Motion Detection
Step 1: Enable Motion Detection
            * Draw detection zones

            * Set sensitivity: 55–65%

            * Set object size: medium

            * Test in live preview to confirm motion triggers

Step 2: Create Event Rule
               * Go to Event → Rule Settings → Conventional

               * Trigger Source: Motion Detection

               * Schedule: Always

               * Attach Trigger Action: Upload to FTP

               * Image upload settings: 1 image per trigger, 1 sec interval

Step 3: Plan / Schedule
                  * Enable Plan → ON

                  * Set Armed for all 24h x 7 days (click drag across grid)

                  * Save Plan

                  * Confirm Rule Settings, Trigger Action, and Motion Detection are all ON

________________


5️⃣ Testing Workflow
                     1. Clear the FTP folder: C:\Users\Guest-UH\Desktop\Geovision_FTP

                     2. Trigger motion in camera view (walk in front)

                     3. Verify images appear in the folder

                     4. FileZilla Server logs should show:

Connection received
User geovision logged in
STOR image.jpg
Transfer complete

                     5. If it fails:

                        * Check FileZilla is running and listening on port 21

                        * Ensure Windows Firewall allows connections

                        * Confirm camera and laptop are on same subnet

                        * Verify motion detection triggers and schedule is armed

________________


6️⃣ Optional Enhancements
                           * Naming Convention:
Use CAM1_%Y-%m-%d_%H-%M-%S.jpg for timestamps

                           * Scheduled Snapshots:
For flood monitoring, a snapshot every 5 minutes is often more reliable than motion triggers

                           * Edge Cases:
Unplug/replug Ethernet to test recovery

                           * Future Modem Integration:
Once PoE or field deployment is ready, the camera can use a Sierra Wireless modem with the same FTP rules for remote uploads
